# Портфолио — инструкция

## Что где лежит

```
index.html              главная страница целиком (вёрстка, стили, скрипты)
start.sh / start.bat    запуск на своём компьютере
README.md               эта инструкция
robots.txt              правила для поисковиков
.htaccess               настройки Apache: заголовки безопасности, запрет доступа к базе
.gitignore              чтобы база заявок не попала в репозиторий

api/config.php          единственный файл, который нужно править: пароль, Telegram, лимиты
api/submit.php          приём заявок с формы
api/albums.php          отдаёт альбомы главной странице
api/admin.php           панель управления

img/                    фотографии: обложка, фоны, портрет
img/.htaccess           запрет исполнения скриптов в папке загрузок — не удаляйте
img/albums/             кадры альбомов, сюда панель кладёт загруженное
assets/favicon.svg      иконка сайта
```

База заявок `leads.sqlite` создастся сама на уровень выше папки сайта.

---

## Запуск на своём компьютере

### Вариант 1 — просто посмотреть вёрстку

Откройте `index.html` двойным щелчком. Всё работает, кроме формы: она
переходит в тестовый режим и честно об этом пишет вместо отправки.
Этого достаточно, чтобы наполнять сайт фотографиями и текстами.

### Вариант 2 — с рабочей формой и админкой

Нужен PHP версии 7.4 или новее. Проверьте: `php -v`

- **Windows** — скачайте PHP с windows.php.net, распакуйте, добавьте папку в PATH
- **macOS** — `brew install php`
- **Linux** — `sudo apt install php-cli php-sqlite3`

Дальше из папки сайта:

```bash
./start.sh          # macOS, Linux
start.bat           # Windows — двойной щелчок
```

Скрипт сам найдёт свободный порт и напечатает адреса. Или вручную:

```bash
php -S localhost:8000
```

- сайт — http://localhost:8000
- админка — http://localhost:8000/api/admin.php
- пароль по умолчанию — `change-me` (замените до публикации)

Встроенный сервер PHP годится только для отладки, в интернет его выставлять нельзя.

## Наполнение контентом

### Фотографии альбомов

Через панель управления: откройте `/api/admin.php`, вкладка **Новый сет** —
укажите название и выберите файлы. Готовые альбомы правятся во вкладке
**Альбомы**: переименовать, добавить кадры, удалить отдельное фото или весь сет.

Главная страница забирает альбомы с сервера сама, ничего в коде править не нужно.
Первая фотография сета становится обложкой.

Если сайт открыт напрямую двойным щелчком (без сервера), показывается
запасной список-заглушка из блока `ALBUMS` внизу `index.html`.

### Фоновые фото

Задаются прямо в разметке, каждое в своём месте:

```html
<header class="cover" style="--cover-photo:url('img/cover.jpg')">
<section id="about" class="about-sec" style="--about-photo:url('img/about-bg.jpg')">
<div class="contact-photo" style="--contact-photo:url('img/contact.jpg')">
```

### Тексты

Заглушки, которые нужно заменить: `Имя Фамилия`, `hello@example.com`,
ссылки `t.me/username`, `vk.com/username`, `instagram.com/username`,
цитата и биография в секции «О себе», описания в «Что я снимаю».

### Подготовка фотографий

- обложки альбомов — под пропорцию 21:9, ширина до 2000 px
- фон заставки — 1920×1080, он всё равно затемняется
- формат WebP: вдвое легче JPEG при том же качестве
- держите вес одного файла до 300 КБ, иначе страница будет открываться долго

## Панель управления

Адрес: `/api/admin.php`. Пароль по умолчанию — `change-me`, смените до публикации.

Три вкладки:

- **Заявки** — что пришло с формы. Конверт отмечает прочитанным, корзина удаляет,
  есть выгрузка всех заявок в CSV.
- **Альбомы** — переименование, добавление кадров, удаление отдельных фото и целых сетов.
- **Новый сет** — название плюс выбор файлов. Загружаются JPEG, PNG и WebP до 8 МБ,
  не больше 60 штук за раз.

Изменения появляются на сайте сразу — главная забирает альбомы через `api/albums.php`.

### Если фотографии не загружаются

Панель пишет причину прямо на экране. Частые случаи:

- *«Не удалось сохранить файл»* — у папки `img/albums` нет прав на запись.
  Поставьте права 755: `chmod 755 img/albums`
- *«Это не изображение»* — файл повреждён либо это не JPEG, PNG или WebP.
- Ничего не происходит при большом количестве файлов — упирается в лимиты PHP.
  В `php.ini` увеличьте `upload_max_filesize`, `post_max_size` и `max_file_uploads`.

## Публикация на хостинге

1. Смените пароль админки. На сервере выполните:
   ```bash
   php -r "echo password_hash('свой-пароль', PASSWORD_DEFAULT);"
   ```
   Результат вставьте в `ADMIN_HASH` в `api/config.php`.

2. Укажите свой домен в `ALLOW_ORIGIN`, например `https://example.ru`.

3. Залейте файлы в корень сайта. Проверьте, что `leads.sqlite`
   не открывается по прямой ссылке — должна быть ошибка 403 или 404.

4. Подключите HTTPS (у большинства хостингов Let's Encrypt бесплатно
   и включается одной кнопкой). После этого раскомментируйте в `.htaccess`
   блоки про HSTS и перенаправление на HTTPS.

### Если сервер на nginx

`.htaccess` там не читается. Добавьте в конфиг сайта:

```nginx
add_header X-Content-Type-Options "nosniff" always;
add_header X-Frame-Options "SAMEORIGIN" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;

location ~* \.(sqlite|sqlite3|db|log|md)$ { deny all; }
location / { autoindex off; }
```

## Что сделано для безопасности

**Форма и база данных**

- все запросы к базе идут через подготовленные выражения — SQL-инъекция невозможна
- база лежит выше папки сайта и скачать её по ссылке нельзя
- скрытое поле-ловушка отсеивает спам-ботов
- не больше 5 заявок с одного IP за час
- ограничение на размер запроса и длину каждого поля
- запросы принимаются только со своего домена

**Админка**

- пароль хранится хешем (bcrypt), в открытом виде его нигде нет
- не больше 5 неудачных попыток входа за 15 минут
- одноразовый токен на каждое действие — чужой сайт не сможет удалить заявки за вас
- идентификатор сессии обновляется при входе
- кука сессии недоступна из JavaScript и не уходит на чужие домены
- весь текст заявок экранируется при выводе — XSS через форму не пройдёт
- страница закрыта от индексации поисковиками

**Загрузка файлов**

- тип определяется по содержимому файла, а не по расширению из браузера
- принимаются только JPEG, PNG и WebP, до 8 МБ
- имя файла придумывает сервер, присланное браузером не используется вообще
- в папке `img` лежит `.htaccess`, запрещающий исполнять там скрипты:
  даже если кто-то сумеет положить туда PHP-файл, он останется просто файлом

**Заголовки**

`X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`,
`Permissions-Policy` — ставятся и через `.htaccess`, и из PHP,
так что работают при любой конфигурации сервера.

### Что стоит сделать дополнительно

- **HTTPS обязателен.** Без него пароль от админки идёт по сети открытым текстом.
- **Смените пароль по умолчанию** до того, как сайт станет доступен извне.
- **Внешние библиотеки.** Шрифты и иконки грузятся с Google Fonts и cdnjs.
  Это безопасно, но у российских провайдеров бывают перебои с доступом
  к этим адресам. Если сайт для российской аудитории — надёжнее скачать
  шрифты и Font Awesome и положить рядом с сайтом. Заодно быстрее загрузка
  и не нужно предупреждение про передачу данных третьим лицам.
- **Персональные данные.** Форма собирает имя и контакт, то есть по закону
  это персональные данные. Для сайта в России нужны политика
  конфиденциальности и галочка согласия рядом с кнопкой отправки.
  Скажите, если нужно — добавлю и то, и другое.
